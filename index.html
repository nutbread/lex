About

This is a set of lexical analizers for language tokenizing. Currently there are libraries for processing JavaScript with source code in JavaScript and Python 2/3.

It was primary written to address some edge case parsing issues found in several major applications (Notepad++, Firefox, Sublime Text, Github/Ace.) These cases usually involve regular expressions or sign-prefixed numbers.

Downloads

  • lexjs.js – JavaScript source, can run on both node.js and browsers
  • lexjs.py – Python source, versions 2.x or 3.x should both work

Usage

Using these libraries is as simple as importing, creating a Lexer, then repeatedly calling the get_token method until it returns null.

get_token will otherwise return a Token object with 3 fields:

  • text – the token string
  • type – the type constant of the token
  • flags – flags for the token
    Most flags are primarily used internally, but are occasionally useful outside the Lexer

JavaScript token type constants:
INVALID, KEYWORD, LITERAL, IDENTIFIER, NUMBER, STRING, REGEX, OPERATOR, WHITESPACE, COMMENT

JavaScript token flags (that can be useful outside the Lexer):
FLAG_MEMBER, // indicates the word is a member (identifier_word.member_word)
FLAG_VERSION_FUTURE, // this keyword is a future reserved keyword
... // Additional token flag constants can be found by opening the library's source

For additional help, view some of these test files, as examples are often more useful than wordy documentation.
